## 3 Historical Evolution

<a id="p-3-historical-evolution-1"></a><!-- para:3-historical-evolution-1 --> The story of LLMs for code is a five-year arc from "can a model autocomplete a line?" to "can an agent resolve a real GitHub issue end to end?" Reading it in order is the fastest way to understand *why* the field looks the way it does: each milestone removed a specific limitation of the previous one. This section is that chronology; later sections dissect the methods each milestone introduced.

<!-- sec:3.1 -->
### <a id="sec-3.1"></a>3.1 The Pre-Generative Era: Code Representation Models

<a id="p-31-the-pre-generative-era-code-representation-models-1"></a><!-- para:31-the-pre-generative-era-code-representation-models-1 --> Before generation came *understanding*. CodeBERT (2020) is an encoder-only model — "exactly the same model architecture as RoBERTa-base," 125M parameters — trained bimodally on paired natural language and programming language <!-- cite:2 --> [[2]](references.md#ref-2). Its objectives were masked language modeling plus replaced-token detection, and it was evaluated on code *search*, documentation generation, and probing — not on functional code generation, for which an encoder is structurally unsuited. CodeBERT (and its cross-modal successor UniXcoder <!-- cite:14 --> [[14]](references.md#ref-14)) established that pretraining transfers to code and produced reusable representations for retrieval, but it could not write a program. The generative breakthrough required a decoder.

<!-- sec:3.2 -->
### <a id="sec-3.2"></a>3.2 The Codex Moment (2021)

<a id="p-32-the-codex-moment-2021-1"></a><!-- para:32-the-codex-moment-2021-1 --> Codex is the field's inflection point. OpenAI fine-tuned a GPT-3-family decoder on code collected in May 2020 from 54 million public GitHub repositories — 179 GB of unique Python files under 1 MB, filtered to a final 159 GB <!-- cite:1 --> [[1]](references.md#ref-1). On the 164-problem HumanEval benchmark it introduced, Codex-12B solved 28.8% of problems with a single sample (pass@1), versus 0% for GPT-3 and 11.4% for GPT-J; allowing 100 samples per problem, *some* sample passed for 70.2% of problems <!-- cite:1 --> [[1]](references.md#ref-1). Three lasting contributions came out of this one paper: a large-scale generative code model (the engine behind GitHub Copilot), the **HumanEval** benchmark, and the **pass@k** metric (Section 4). Codex reframed code generation as a functional-correctness problem rather than a text-matching one — the decision that shaped everything after.

<!-- sec:3.3 -->
### <a id="sec-3.3"></a>3.3 The Open-Model Wave (2022–2024)

<a id="p-33-the-open-model-wave-20222024-1"></a><!-- para:33-the-open-model-wave-20222024-1 --> Codex was closed. The next two years were defined by open models that progressively matched and then exceeded it, while opening every part of the recipe.

- <a id="p-33-the-open-model-wave-20222024-2"></a><!-- para:33-the-open-model-wave-20222024-2 --> **CodeGen (2022)** delivered the first fully open competitive family (350M–16.1B parameters), trained sequentially on text then multilingual code then Python, and introduced multi-turn program synthesis; CodeGen-Mono-16.1B reached 29.28% HumanEval pass@1 <!-- cite:3 --> [[3]](references.md#ref-3).
- **InCoder (2022)** was the first large generative code model that could *infill* — generate a span conditioned on both left and right context — via a causal-masking objective, unlocking editing, type inference, and comment generation as zero-shot tasks <!-- cite:4 --> [[4]](references.md#ref-4).
- **The fill-in-the-middle paper (Bavarian et al., 2022)** showed infilling could be added to any autoregressive model essentially for free, by a simple document transformation (Section 4), making FIM a standard ingredient <!-- cite:5 --> [[5]](references.md#ref-5).
- **Code Llama (2023)** built a strong open family (7B–70B) on Llama 2 via a specialization pipeline (+500B code tokens, Python specialization, long-context fine-tuning to 100k tokens, instruction tuning); Code Llama-Python-70B reached 57.3% HumanEval pass@1 and the Instruct-70B variant 67.8% — competitive with the then-current GPT-4 number of 67.0% reported in the same table <!-- cite:6 --> [[6]](references.md#ref-6).
- **StarCoder (2023)** paired an open model with an open, *governed* dataset (The Stack), 15B parameters trained on 1T tokens with 8k context, FIM, and multi-query attention for fast inference <!-- cite:7 --> [[7]](references.md#ref-7), <!-- cite:8 --> [[8]](references.md#ref-8).
- **DeepSeek-Coder (2024)** trained from scratch on 2T tokens across 87 languages with repository-level pretraining and FIM, and its instruct variant surpassed GPT-3.5-Turbo on code tasks <!-- cite:10 --> [[10]](references.md#ref-10).
- **Qwen2.5-Coder (2024)** formalized the modern three-stage recipe — file-level pretraining (5.2T tokens), repository-level long-context pretraining (300B tokens, 128k window), then SFT + DPO alignment <!-- cite:11 --> [[11]](references.md#ref-11).

<a id="p-33-the-open-model-wave-20222024-3"></a><!-- para:33-the-open-model-wave-20222024-3 --> By mid-2024, open models had effectively closed the function-level gap to frontier systems. DeepSeek-Coder-V2 (2024), a 236B-total / 21B-active mixture-of-experts model over 338 languages, was explicitly framed as "breaking the barrier of closed-source models," reaching GPT-4-Turbo-comparable code performance with open weights <!-- cite:43 --> [[43]](references.md#ref-43).

<!-- sec:3.4 -->
### <a id="sec-3.4"></a>3.4 The Reasoning and Agentic Turn (2024–2026)

<a id="p-34-the-reasoning-and-agentic-turn-20242026-1"></a><!-- para:34-the-reasoning-and-agentic-turn-20242026-1 --> Two shifts then redefined the frontier. First, **reasoning**: DeepSeek-R1 (2025) showed that competitive-programming-level ability can *emerge from reinforcement learning with verifiable, rule-based rewards alone* — no supervised reasoning traces required — reaching the 96.3rd percentile on Codeforces (Section 9) <!-- cite:25 --> [[25]](references.md#ref-25). OpenAI's report on competitive programming with large reasoning models made the same point for its o-series, with general RL scaling overtaking hand-built inference pipelines <!-- cite:26 --> [[26]](references.md#ref-26). The free, dense correctness signal that makes code special (Section 2) turned out to be exactly the reward such methods need.

<a id="p-34-the-reasoning-and-agentic-turn-20242026-2"></a><!-- para:34-the-reasoning-and-agentic-turn-20242026-2 --> Second, **agency**: the unit of work moved from "a function" to "a task in a repository." SWE-bench (2023) reframed evaluation around resolving real GitHub issues, and on its debut the best model resolved a mere 1.96% <!-- cite:38 --> [[38]](references.md#ref-38). SWE-agent (2024) then showed that a purpose-built agent-computer interface — not just a bigger model — drives most of the gain (Section 12) <!-- cite:32 --> [[32]](references.md#ref-32). By late 2025, frontier agentic systems report resolving roughly 80% of SWE-bench Verified, albeit with documented contamination caveats (Section 15). The field's center of gravity is now the autonomous coding agent — the natural endpoint of a modality whose ground truth was executable all along.
