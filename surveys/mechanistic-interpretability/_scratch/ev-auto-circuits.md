## Q1: ACDC — Automatic Circuit DisCovery (Conmy et al. 2023, arXiv:2304.14997) — the recursive edge-pruning algorithm, the metric it optimizes, and how it was evaluated against hand-found circuits.

- **Finding:** ACDC (Automatic Circuit DisCovery) is a greedy, deterministic algorithm that starts from the full computational graph of a transformer (nodes = model components — attention heads, MLPs — edges = the residual-stream data paths between them) and prunes edges one at a time in reverse-topological order (from output back to input), permanently removing an edge if its removal changes a chosen behavioral metric by less than a threshold τ. It was evaluated against hand-found "canonical" circuits (IOI, Greater-Than, Docstring, tracr-compiled toy tasks, Induction) via ROC-style precision/recall on whether the discovered edge/node set matches the human-identified circuit, and compared to two baselines, Subnetwork Probing (SP) and Head Importance Score for Pruning (HISP).
- **Mechanism / derivation notes:** Algorithm (from the paper's Algorithm 1, as reconstructed from ar5iv full text): (1) Let G be the full computational graph, H := G initially. (2) Reverse-topologically sort all nodes v (i.e., process consumers before their producers, output-to-input). (3) For each node v, for each parent/incoming edge from w → v (in the order the edges feed v): temporarily delete edge (w→v) from the current hypothesis graph H, producing H_new; compute the metric degradation D_KL(G || H_new) − D_KL(G || H) (the paper uses KL divergence between the full model's output distribution and the pruned subgraph's output distribution, evaluated on a fixed prompt distribution, as the default metric, though other differentiable metrics — e.g. logit difference — can substitute); (4) if this metric increase is below threshold τ, permanently remove the edge (keep H_new as the new H); otherwise keep the edge and restore H. (5) Continue until all nodes/edges are processed; return the final pruned subgraph H as the discovered circuit. Intuition: an edge is causally load-bearing for the behavior iff ablating it (replacing its contribution with the value it would take under a reference/corrupted distribution) meaningfully changes the metric; ACDC operationalizes "meaningfully" via the τ threshold and searches greedily rather than exhaustively over the exponential edge-subset space, exploiting the DAG structure (reverse-topological order) so that an edge's removal is evaluated in the context of already-decided downstream removals.
- **Values:** Full GPT-2 Small computational graph ≈ 32,000 edges; ACDC's discovered circuit on the full multi-task setting selected 68 of the 32,000 edges, "all of which were manually found by previous work" (quote via search synthesis of ar5iv text). On Greater-Than, ACDC rediscovered 5/5 of the component types of the hand-found circuit. On the IOI task, the hand-found ("canonical") circuit is reported (via secondary source) as containing 1,041 edges; ACDC's edge-level ROC/AUC on IOI and Greater-Than and a tracr-reverse task exceeded both Subnetwork Probing and HISP's AUC. On tracr-compiled tasks (toy transformers implementing a known ground-truth algorithm), ACDC "fully recovers the circuit... at any threshold τ>0" (exact ground truth available since tracr compiles the algorithm directly). Paper published at NeurIPS 2023 (Spotlight); v1 submitted 2023-04-28, v4 revised 2023-10-28. Authors: Arthur Conmy, Augustine N. Mavor-Parker, Aengus Lynch, Stefan Heimersheim, Adrià Garriga-Alonso.

## Q2: Edge Attribution Patching (EAP) and EAP-IG (Syed et al. 2023, arXiv:2310.10348; Hanna et al. 2024, arXiv:2403.17806, "Have Faith in Faithfulness") — gradient-based scalable circuit discovery and the faithfulness improvement of the integrated-gradients variant.

- **Finding:** EAP replaces ACDC's per-edge forward-pass ablation with a single first-order (gradient) linear approximation of each edge's causal effect, computed from just two forward passes (clean + corrupted) and one backward pass, making circuit discovery over the full edge set tractable in one shot rather than iteratively; Syed et al. (2023) show EAP achieves greater AUC for circuit recovery than ACDC/SP/HISP baselines while being far cheaper. However, Hanna et al. (2024) show plain EAP's linear approximation saturates (near-zero gradients on some tasks) and so recovers circuits with high *node overlap* to hand-found circuits but poor *faithfulness*; their fix, EAP-IG, integrates the gradient along a path from corrupted to clean activation (an integrated-gradients-style path integral) rather than using the single-point gradient at the corrupted input, which substantially improves faithfulness while only mildly affecting node overlap.
- **Mechanism / derivation notes:** EAP scores each edge u via a first-order Taylor approximation of the effect of patching (substituting) the upstream activation on the downstream metric: attribution ê_u = (a_u(x′) − a_u(x))ᵀ · ∂L(x)/∂a_u, where a_u(·) is the activation flowing along edge u under clean input x vs corrupted/counterfactual input x′, and L is a scalar differentiable metric (e.g., logit difference) evaluated at the clean run. This requires only one gradient (∂L/∂a_u at the clean activations) and the two forward-pass activation values (clean and corrupted), so all edges' attributions are obtained from exactly 2 forward passes + 1 backward pass rather than ACDC's per-edge forward pass. Edges are then ranked/thresholded by |ê_u| to select the circuit (a single ranking pass, not ACDC's iterative reverse-topological loop). Weakness: this is a linearization at a single point (the clean activation, using the gradient there); if the true function is highly nonlinear between x and x′ (e.g. saturates, or has near-zero gradient in a flat region near the corrupted point) the single-point gradient badly mis-estimates the true causal (patched) effect — this is the "gradient saturation" failure mode. EAP-IG fixes this by replacing the single-point gradient with an integrated-gradients-style path integral: it samples m interpolated activations along the straight-line path from a_u(x′) to a_u(x) (i.e., a_u(x′) + k/m·(a_u(x) − a_u(x′)) for k=0..m), evaluates the gradient of L at each interpolated point, and averages — this is the standard Integrated Gradients construction (Sundararajan et al. 2017) applied per-edge in the EAP setting, so that even if the gradient is near zero exactly at x′ or x, the path samples pick up the true steep transition region in between. Faithfulness (the property actually being optimized for, as argued by Hanna et al.) is defined operationally: a circuit is faithful if ablating (patching to corrupted activations) all edges/nodes *outside* the circuit leaves task performance (the metric) unchanged relative to the full model — i.e., faithfulness is measured as the fraction of full-model performance recovered by the circuit-only subgraph under the corrupted-everything-else ablation, NOT as node/edge overlap with a hand-found circuit. The paper's central methodological claim: circuit-comparison work should report faithfulness curves (performance recovered vs. circuit size) rather than just overlap with a previously-published circuit, because two circuits can have high node overlap and still differ hugely in faithfulness.
- **Values:** EAP: "just two forward passes and a backward pass" per full-graph attribution computation (vs. ACDC's O(edges) forward passes); "averaged over all tasks, [EAP] has greater AUC for circuit recovery than other methods" (Syed et al. 2023 abstract-level claim). EAP-IG faithfulness numbers (Hanna et al. 2024): on the Greater-Than (GT) task, "the smallest EAP circuit fulfilling the criterion of containing at least 90% of manually-found circuit nodes achieves only 51% faithfulness"; on IOI, "the smallest EAP circuit that contains at least 90% of the manually-found IOI circuit's nodes achieves 0% faithfulness" — i.e., despite matching 90%+ of the "correct" nodes, the EAP circuit at that size is not causally sufficient at all on IOI. EAP-IG circuits are reported as more faithful than EAP circuits "on many tasks," though activation-patching (ground-truth, expensive) circuits still sometimes outperform both EAP and EAP-IG in faithfulness. Country-Capital task cited as an example where EAP-IG's intermediate-point sampling captures a "steep transition" that plain EAP misses.

## Q3: Causal Scrubbing (Chan, Garriga-Alonso, Goldowsky-Dill, Greenblatt, Nitishinskaya, Radhakrishnan, Shlegeris, Thomas 2022, Redwood Research, AI Alignment Forum / LessWrong) — hypothesis-as-resampling-ablation, what it validates, and its known limitations.

- **Finding:** Causal scrubbing (CaSc) is a method for rigorously testing a mechanistic-interpretability hypothesis by converting the informal hypothesis ("these nodes/edges compute this abstract quantity, and everything else is causally irrelevant to it") into a formal correspondence between the model's real computational graph and a human-interpretable computational graph, then recursively performing resampling ablations — replacing each non-hypothesized-relevant activation with its value on a different, hypothesis-equivalent input — and checking whether behavior (task loss/metric) is preserved. If the "scrubbed" model's loss is significantly worse than the original model's loss, the hypothesis is falsified; if scrubbed loss ≈ original loss, the hypothesis survives that test (though CaSc cannot prove a hypothesis fully correct, only fail to falsify it). It was applied by Redwood Research to refine understanding of induction heads in a small language model and of how an algorithmic transformer classifies parenthesis-balancing.
- **Mechanism / derivation notes:** (1) Hypothesis formalization: a human specifies a correspondence between the model's computational graph G (real nodes = attention heads/MLPs/embeddings) and an interpretable/abstract computational graph H (the claimed algorithm, e.g. "previous-token head → induction head → output"), where each node in H maps to one or more nodes in G, and edges in H specify which values are claimed to matter functionally. (2) Resampling ablation: for every real node NOT claimed by the hypothesis to matter for a given downstream computation, causal scrubbing replaces its activation with the activation that node would have taken on a *different* input x′ that is equivalent to the original input x under the hypothesis' definition of "equivalent" (i.e., x and x′ agree on whatever the hypothesis says should matter, but may differ arbitrarily on everything else) — this is an interchange intervention, but chosen via (recursive) rejection sampling over the dataset to find valid x′ for each node in the graph, applied recursively from the output backward through the correspondence tree so that upstream resamplings are nested consistently with downstream ones. (3) Metric: "scrubbed loss" = the model's loss (or task metric) after this resampling is applied throughout the graph, evaluated over many sampled input/reference pairs. (4) Falsification criterion: if scrubbed loss is significantly higher (worse) than the original unscrubbed loss, the hypothesis is wrong (it claimed some resampled-away computation didn't matter, but empirically it did) — this is a strict, asymmetric test: CaSc can definitively reject a false hypothesis (when the resampling breaks behavior) but cannot fully confirm a true one (a hypothesis could pass CaSc's test yet still not be the actual full mechanism, since CaSc doesn't test whether the specification is *complete* or *unique*, only whether it's *sufficient/consistent*). Two known failure modes for the falsification side itself: (a) "cancellation" — a false positive where the resampling makes the model do better on some inputs and worse on others such that the *average* scrubbed loss looks unchanged even though the hypothesis is actually wrong at the individual-input level; (b) CaSc cannot check hypothesis *specificity* — it cannot distinguish between two hypotheses that are extensionally equivalent under the intervention distribution used, nor detect a misidentified feature (e.g., calling a component an "A-detector" when it's actually a "B-detector" if A and B are perfectly correlated in the data used for resampling).
- **Values:** Induction-head case study: a naive/narrow hypothesis about how induction heads work explained only ~35% of the (recoverable) loss, while a more general/refined hypothesis explained ~89% of the loss. More granular ablation-by-ablation breakdown reported for the induction case: scrubbing all input to the Query except the token embeddings retained 51% of loss recovered; scrubbing input to the Key except the previous-token-head contribution retained 79%; scrubbing input to the Value except embeddings retained 90%. IOI-task application is described (via secondary source) as producing "mediocre results" relative to the induction case, i.e. the simple hand-found IOI circuit does not fully pass causal scrubbing — exact IOI percentage not found in the sources surfaced. Original post published 2022-12-03 on the AI Alignment Forum (cross-posted LessWrong).

## Q4: Distributed Alignment Search (DAS) (Geiger, Wu, Potts, Icard, Goodman 2023/2024, arXiv:2303.02536; Boundless DAS, Wu et al. 2023, arXiv:2305.08809) — learning an orthogonal subspace where interchange interventions align with an abstract causal variable.

- **Finding:** DAS is a method for testing/finding whether an abstract, human-interpretable causal variable (from a hypothesized high-level causal model) is linearly encoded in a *distributed* subspace of a neural network's hidden representation — rather than assuming it corresponds to individual neurons or a human-guessed direction, DAS learns an orthogonal rotation of the representation space via gradient descent such that intervening on a fixed-size subset of the rotated coordinates (an interchange intervention swapping those coordinates' values from a "source" input into a "base" input, then rotating back) reproduces the high-level causal model's predicted output as closely as possible. This replaces prior causal-abstraction work's brute-force search over neuron subsets (and the assumption that high-level variables map onto *disjoint* neuron sets) with a differentiable, learned alignment. Boundless DAS (Wu et al. 2023, NeurIPS 2023) extends DAS to scale to real LLMs (Alpaca) by additionally learning the *size* of the intervened subspace (via differentiable soft boundary/mask parameters) rather than fixing it a priori, and using this to discover that Alpaca (a 7B instruction-tuned LLaMA variant) implements an entity-tracking behavior via two identifiable interpretable boolean variables in specific attention-head representations.
- **Mechanism / derivation notes:** Setup: a high-level causal model M_H with an intermediate variable v that the researcher hypothesizes is computed somewhere inside the network; a low-level neural network M_L with hidden representation h at some layer/position. DAS learns an orthogonal matrix R (Rᵀ R = I) that defines a rotated basis for the representation space, R·h; a fixed-dimensionality subset of the rotated coordinates (a linear subspace, spanned by some columns of R) is designated as "where v lives." A *distributed interchange intervention* is then: take a "base" input's hidden state h_base and a "source" input's hidden state h_source, compute Rh_base and Rh_source, replace the designated k coordinates of Rh_base with those of Rh_source (leaving the other d−k coordinates from h_base unchanged), then rotate back via Rᵀ to get the intervened hidden state, and continue the forward pass with that intervened representation substituted at that layer/position. R is trained (via gradient descent) to *maximize Interchange Intervention Accuracy (IIA)* — the fraction of (base, source) pairs for which the network's output after this intervention matches what the high-level causal model M_H predicts should happen when the corresponding high-level variable v is analogously set to its source-input value. High IIA (approaching 100%) after training R means the causal hypothesis (that v is linearly encoded in that subspace, in that rotated basis) is a faithful description of the network's actual computation at that site; DAS effectively performs supervised metric learning of the alignment, replacing the earlier "geometric" or brute-force causal-abstraction search over neuron combinations (whose combinatorics are intractable at LLM scale) with one gradient-trained rotation per hypothesized site. Boundless DAS extension: to avoid manually choosing subspace size k, it introduces a learnable soft "boundary" per coordinate — two learnable boundary scalars and a temperature-scaled sigmoid function produce a soft mask over the rotated dimensions (mask(i) ≈ 1 for i inside the learned boundary, ≈0 outside), which is optimized jointly with R via the same IIA objective; as training proceeds the mask sharpens toward a binary indicator, effectively learning both *where* (rotation) and *how large* (boundary/size) the causal subspace is, which is what makes it tractable to search many candidate sites/layers in a large model (Alpaca) without hand-picking dimensionality at each site.
- **Values:** DAS original paper (Geiger et al. 2023/2024, CLeaR 2024 proceedings): experiments include synthetic tasks such as hierarchical equality — determine whether (w = x) = (y = z) given two pairs of objects — used to test whether interchange interventions on a learned rotated subspace of a 3-layer ReLU feed-forward network's hidden layer recover the compositional structure; a related/adjacent finding surfaced in search (attributed generally to this causal-abstraction-alignment line of work, not pinned to a single exact paper/table in this search pass) reports interchange intervention accuracy "over 80%" using non-linear alignment maps in randomly-initialized (untrained) networks — flagged as needing direct re-verification against the specific paper/table before use as a headline DAS number. Boundless DAS / Alpaca (Wu et al. 2023, NeurIPS 2023): Boundless DAS discovered that Alpaca (7B) implements its entity-tracking behavior via a causal model with two interpretable boolean variables, located in specific attention-head representations, at "a fraction of the cost" of the original brute-force DAS search (paper abstract-level claim; exact wall-clock/compute reduction factor and exact IIA numeric value for the discovered Alpaca alignment were not pinned down within the 5-search budget for this question).
