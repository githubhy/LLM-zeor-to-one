## Q1: Activation patching / interchange interventions / causal tracing — clean vs corrupt runs, denoising vs noising, logit-difference/probability metric, intervention formalism

- **Finding:** Activation patching (a.k.a. causal tracing / interchange intervention) runs a model twice — a "clean" run on the original prompt and a "corrupted" run where some input tokens are perturbed (e.g. subject-token embeddings noised) — then re-runs the corrupted computation while surgically copying ("patching"/"restoring") one internal activation from the clean run into the corrupted run at a specific (layer, token position). The change in the output metric (probability of the correct answer, or the logit difference between correct/incorrect answer) attributable to that single patched activation is the causal-tracing/patching effect at that site, and sweeping the patch site over all (layer, token) pairs produces a 2D "causal trace" heatmap that localizes which components carry the causally relevant information.
- **Mechanism / derivation notes:**
  - Two runs: clean run on prompt $x$ gives hidden states $\{h_l^{(t)}\}$ and output $P(y \mid x)$. Corrupted run: perturb the subject-token embeddings by adding Gaussian noise $\epsilon \sim \mathcal N(0, \nu^2 I)$ (ROME sets $\nu$ = 3× the empirical standard deviation of the model's embedding matrix), giving corrupted output $P^*(y\mid x)$ with a degraded probability of the correct answer.
  - "Corrupted-with-restoration" run: run the corrupted computation, but at one chosen (layer $l$, token $t$) location, overwrite the corrupted activation $h_l^{*(t)}$ with the clean activation $h_l^{(t)}$ from the clean run, then let the forward pass continue unmodified afterward. Record the resulting output probability $P^{*,\, \mathrm{clean}\, h_l^{(t)}}(y\mid x)$.
  - **Total effect (TE)** = $P(y\mid x) - P^*(y\mid x)$ (clean minus fully-corrupted probability of the correct answer) — the overall damage done by corruption.
  - **Indirect effect (IE)** at a given (layer, token) = $P^{*,\,\mathrm{clean}\,h_l^{(t)}}(y\mid x) - P^*(y\mid x)$ — how much of the total effect is recovered by restoring just that one activation; this is the causal-tracing score plotted at that grid cell.
  - This is a special case of Pearl-style causal mediation analysis (mediators = internal activations; treatment = corruption; outcome = model's output probability), following Vig et al. (2020)'s adaptation of causal mediation analysis to transformer LMs (treatment = swapping a pronoun/gendered cue; mediators = individual neurons/attention heads; direct effect = the change in output not mediated by a given internal unit, indirect effect = the change mediated through it).
  - **Denoising vs noising direction** (terminology formalized in Zhang & Nanda 2023, arXiv:2309.16042): (a) **Denoising** = patch a clean activation into an otherwise-corrupted run (clean → corrupt patch, i.e. ROME's "restoration" direction above); a large recovery of the clean behavior shows the patched activation was **sufficient** to restore correct behavior, implying it is part of a sufficient cross-section of the circuit. (b) **Noising** = the reverse — patch a corrupted activation into an otherwise-clean run (corrupt → clean patch); a large degradation of clean behavior shows the patched activation was **necessary** to maintain correct behavior. The two directions are asymmetric and can give different circuit conclusions on the same model/task.
  - **Metric choice:** the two dominant output metrics are (i) probability of the correct-answer token(s) (used by ROME) and (ii) **logit difference** between a correct-answer logit and a designated incorrect/distractor-answer logit (used by IOI-style circuit work); Zhang & Nanda (2023) argue logit difference is the superior metric because it has finer-grained control (isolates the specific correct-vs-incorrect contrast) and can detect "negative" components (components that actively suppress the correct answer), which a pure probability metric can mask due to softmax saturation/normalization effects.
  - Corruption method also varies: **Gaussian noising (GN)** (ROME's ε-noise on embeddings) vs **symmetric token replacement (STR)** (swap in a different, real, in-distribution prompt as the "corrupted" input, e.g. IOI's swap of names) — Zhang & Nanda recommend STR because it keeps the corrupted input in-distribution, whereas GN can push activations off-manifold and produce misleading effects.
- **Values:**
  - ROME Gaussian-noise scale: $\nu = 3\sigma_{\mathrm{emb}}$ where $\sigma_{\mathrm{emb}}$ is the empirical standard deviation of the model's embedding vectors — "In ROME's causal tracing, noise is sampled from a zero-centered spherical Gaussian, whose standard deviation is 3 times the σ standard deviation of the model's embeddings" (paraphrase of arXiv:2202.05262 Sec. 3 methodology, per search-result summary of the paper text; exact equation not independently re-derived from a direct PDF read in this pass — flag for verification against the PDF).
  - No single universal numeric threshold for "large" restoration effect was found in the search snippets (paper reports per-layer/per-token IE curves peaking at middle MLP layers for the subject's last token — qualitative finding, not a single scalar).

## Q2: Path patching (Wang et al. 2022 IOI, arXiv:2211.00593; Goldowsky-Dill et al. 2023, arXiv:2304.05969)

- **Finding:** Path patching generalizes activation patching from patching a single node's activation to patching a single **edge** (or a small set of edges) in the model's computational graph — i.e. the direct causal effect a specific upstream component ("sender") has on a specific downstream component ("receiver") via one connection, with every other computational path held fixed at its clean value. This isolates whether behavior is mediated *through that specific edge* rather than merely through "some path via that node," letting researchers build a directed-graph circuit (e.g. IOI's 26-head, 7-class circuit in GPT-2 small) edge by edge instead of only node by node.
- **Mechanism / derivation notes:**
  - Path patching requires **three forward passes** (vs. activation patching's two):
    1. **Clean run** on the clean/baseline input $x_{\mathrm{clean}}$: cache every activation.
    2. **Corrupted run** on the corrupted input $x_{\mathrm{corrupt}}$: cache every activation.
    3. **Patched run**: run the model on $x_{\mathrm{clean}}$ again, but (a) the designated **sender** node's output is overwritten with its value from the *corrupted* run (step 2), and (b) every other node that is **not** downstream of the sender along the path being tested is held at its *clean* value (step 1) — effectively "freezing" the rest of the model so the corrupted sender's effect can only propagate to the **receiver** node along the one specified path/edge. The receiver's activation (or the final output metric) is then read off this patched run.
  - Contrast with plain activation patching: plain patching swaps one *node's* activation and lets the perturbation propagate through *every* downstream path (all edges out of that node); path patching additionally holds every off-target edge fixed at clean, isolating the causal contribution of one specific sender→receiver edge. This is why path patching needs a third run — the extra run is exactly what "holds everything else clean" while letting the one edge carry the corrupted value forward.
  - Iterating this procedure (patching each candidate edge one at a time, or in specified sets) is how Wang et al. (2022) built up the IOI circuit graph node-class by node-class: identify heads whose direct path to the logits (a "receiver" = final logit difference) matters (**Name Mover Heads**, e.g. heads 9.6, 9.9, 10.0 — attend to the Indirect-Object token and copy its identity to the output, directly and positively driving the logit difference), then trace backward to heads that feed *those* heads' query/key/value inputs via specific edges (**S-Inhibition Heads** — write to Name Mover queries to suppress attention to the duplicated Subject name; **Duplicate Token Heads** and **Induction Heads** — feed S-Inhibition heads by detecting the repeated Subject token), assembling a 7-class, 26-head directed circuit for the indirect-object-identification task in GPT-2 small.
  - Goldowsky-Dill et al. (2023) formalize this as testing hypotheses of the form "behavior is localized to this set of paths in the computational graph," generalizing Wang et al.'s ad hoc IOI procedure into a reusable framework/algorithm (and open-source library) for expressing and quantitatively testing such path-localization hypotheses, and use it to refine the mechanistic explanation of induction heads and characterize another GPT-2 behavior.
- **Values:** IOI circuit: 26 attention heads across 7 functional classes in GPT-2 small (Name Mover, Negative Name Mover, S-Inhibition, Duplicate Token, Induction, Previous Token, Backup Name Mover — per Wang et al. 2022); example Name Mover Heads at layer.head positions 9.6, 9.9, 10.0 (GPT-2 small has 12 layers × 12 heads). Evaluated via three quantitative criteria: faithfulness, completeness, minimality (exact numeric thresholds/scores for these criteria not captured in the search snippets this pass — flag for direct-PDF follow-up if needed for the survey's quantitative claims).

## Q3: Attribution patching (Nanda 2023 blog; Syed, Rager, Conmy 2023, arXiv:2310.10348)

- **Finding:** Attribution patching is a gradient-based, first-order Taylor approximation to activation patching that estimates the patching effect of *every* activation (or edge) in a single clean forward pass + single corrupted forward pass + single backward pass (three passes total, independent of the number of sites being tested), versus vanilla activation patching's requirement of one extra forward pass *per site* patched. This makes it tractable to approximately localize behavior across every neuron/head/edge in very large models (Nanda's example: all 4.7M neurons of GPT-3 in three passes) but the linear approximation systematically breaks down at specific transformer nonlinearities (softmax saturation in attention patterns; cancellation between direct and indirect effects), producing false negatives that later work (AtP*) specifically patches.
- **Mechanism / derivation notes:**
  - Setup: let $L(x)$ be the scalar output metric (e.g. logit difference) as a function of some internal activation $a$, evaluated at the clean activation $a_{\mathrm{clean}}$ and corrupted activation $a_{\mathrm{corrupt}}$. Vanilla activation patching computes the *exact* patched-metric change $L(a_{\mathrm{corrupt}}) - L(a_{\mathrm{clean}})$ (or vice versa, denoising/noising direction) by literally re-running the network with $a$ swapped to the other run's value at that site — this requires a distinct forward pass for each site.
  - Attribution patching instead takes a **first-order Taylor expansion** of $L$ around the clean (or corrupted) activation: treat the corrupted run as a small, locally linear perturbation of the clean run, so the patching effect of swapping activation $a$ is approximated as

    $$\hat{\Delta L}(a) \approx \nabla_a L(a_{\mathrm{clean}}) \cdot (a_{\mathrm{corrupt}} - a_{\mathrm{clean}})$$

    i.e., the dot product of (i) the gradient of the metric with respect to that activation (computed once via a single backward pass on the clean run, reused for every site) and (ii) the actual clean-minus-corrupted activation difference at that site (available from the two cached forward passes). Because the gradient is a single tensor covering all activations simultaneously, this yields an approximate patching effect for **every** site in the network from just 2 forward passes (clean + corrupted, to get both activation values) + 1 backward pass (to get the gradient) — no per-site extra pass.
  - This is explicitly distinguished from plain input-times-gradient saliency: instead of multiplying the gradient by the raw activation value, attribution patching multiplies by the clean-minus-corrupted *difference*, so the score reflects the task-relevant counterfactual change rather than the absolute activation magnitude.
  - Syed, Rager & Conmy (2023) extend this from patching single nodes/activations to patching **edges** in the computational graph (**Edge Attribution Patching**, EAP) — applying the same linear approximation to estimate the importance of every edge in the model's subgraph, then pruning the least-important edges to automatically discover a sparse circuit; this reduces automated-circuit-discovery cost from needing one forward pass per edge evaluated (as in prior methods like ACDC, so cost grows with model/graph size) down to a small constant number of passes (2 forward + 1 backward), making automated circuit discovery tractable on larger models. Their results show EAP matches or outperforms prior automated circuit-discovery methods (e.g. ACDC) at a small fraction of the compute.
  - **Where the linear approximation fails:** the approximation is only as good as the local-linearity assumption; two systematic failure modes are named in follow-up work: (1) **saturation** — nonlinearities with locally-flat gradients (most notably softmax in attention patterns, which saturates near 0 or 1) give a near-zero gradient even when the *true* patched effect (a discrete jump in attention allocation) is large, producing false negatives (a genuinely important component/edge scored as unimportant); (2) **cancellation** — a node's direct and indirect effects on the metric can partially cancel in the true nonlinear computation in a way the first-order linear sum does not capture, again masking true importance. Empirically, the approximation "works well when patching small activations like head outputs, but performs poorly when patching large activations like a residual stream" (per Nanda's blog summary). AtP* (Kramár et al. 2024, arXiv:2403.00745) is the direct follow-up fix: it patches the specific QK (query-key) computation exactly (recomputing the real attention-pattern change rather than relying on the softmax's local gradient) to fix the saturation failure, while retaining attribution patching's scalability, and adds variance-reduction techniques for the cancellation-sensitive cases.
- **Values:**
  - Nanda's headline scalability claim: "in three passes, you could patch all 4.7M of GPT-3's neurons" (verbatim from the X/Twitter post announcing the blog, March 2023) — i.e. attribution patching needs exactly 2 forward + 1 backward pass regardless of the number of sites (here, 4.7M neurons) being scored, vs. activation patching's ~4.7M separate forward passes for the same coverage.
  - Syed, Rager & Conmy (2023): EAP requires "just two forward passes and a backward pass," contrasted with prior automated circuit discovery (e.g. ACDC) needing one forward pass per edge evaluated, so cost scales with the number of edges/model size — exact FLOP or wall-clock multiplier not captured in the search snippets this pass (flag for follow-up if the survey wants a precise speedup factor on a specific model/edge-count).
