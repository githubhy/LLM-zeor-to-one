## Q1: Linear probing / diagnostic classifiers — what a probe licenses, and the causal critique

- **Finding:** A probe is a classifier trained *post hoc* on a frozen model's intermediate activations to predict some property of interest (POS tag, syntax tree depth, etc.); high probe accuracy licenses only the claim that the property is *linearly (or probe-family-)* decodable from that layer's representation, not that the model's forward computation actually *reads out and uses* that information downstream. Hewitt & Liang (2019) show naive probe accuracy is confounded by the probe's own capacity to memorize/learn the task from the representation regardless of whether the representation encodes the property "naturally," and propose control tasks + a **selectivity** metric (linguistic-task accuracy minus a random control-task accuracy) to strip this out.
- **Mechanism / derivation notes:**
  - Alain & Bengio (2016): train a linear classifier $g_\theta$ on frozen hidden state $h^{(l)}$ at layer $l$ to predict label $y$ (e.g., MNIST class from a hidden layer of a CNN); probe accuracy vs. layer depth is used as evidence for how "information for the task" becomes linearly separable across depth. Framed as a way to give an information-theoretic/geometric read of what each layer represents, motivating design choices like auxiliary (deeply-supervised) heads.
  - Hewitt & Liang (2019) control-task construction: fix a task where inputs (e.g., word types) are mapped to *randomly assigned* structureless outputs; a probe can only solve this by memorizing input→output mappings in its own weights, since there is no linguistic regularity to exploit. **Selectivity** = (accuracy on real linguistic task) − (accuracy on control task with same probe family/capacity). A probe family that has high accuracy on both the real task and the random control task has just learned to memorize an arbitrary mapping (low selectivity ⇒ the probe is doing the work, not the representation); high accuracy + low control accuracy (high selectivity) is the signature that the representation itself carries the structure. They argue simpler probes (e.g., linear, or small MLP with regularization) tend to have higher selectivity than large/high-capacity MLP probes, which can fit almost any control task.
  - Causal critique (generalized, e.g. as systematized in Belinkov 2022 "Probing Classifiers: Promises, Shortcomings, and Advances"): probing is fundamentally correlational — it establishes the property is *encoded* (decodable) somewhere in the representation, not that the property is *used* by the network's actual computation to produce its output. This is exactly the gap that motivates causal-intervention methods (activation patching / causal tracing / ablation) elsewhere in the mech-interp literature — a probe finding "X is decodable at layer 5" is not evidence that the circuit computing the final output actually routes through that encoding of X.
- **Values:** Hewitt & Liang (2019) selectivity metric defined as: `selectivity = (probe accuracy on linguistic task) − (probe accuracy on control task)`; qualitative result: linear probes had *higher selectivity* than MLP-1 probes on their POS-tagging control-task experiments (MLP probes achieve near-ceiling accuracy even on the random control task, i.e., low selectivity), per the paper's summary (arXiv:1909.03368, EMNLP-IJCNLP 2019, pp. 2733-2743, Best Paper Runner-Up). Alain & Bengio (2016): no single headline number extracted in this pass (arXiv:1610.01644) — would need PDF read for the exact per-layer MNIST/ImageNet probe-accuracy curve.

## Q2: Logit lens and Tuned Lens — mechanism, affine-probe fix, key findings

- **Finding:** The logit lens (nostalgebraist, 2020, LessWrong) decodes an intermediate residual-stream state at any layer $l$ by applying the model's own final LayerNorm and unembedding matrix $W_U$ (the same ones used at the output) as if that layer's residual stream were already the final one, yielding a "pseudo-logit" distribution over the vocabulary at every layer. This works well on GPT-2 (predictions converge roughly monotonically toward the final answer across depth) but is **systematically biased** on other model families (GPT-Neo, BLOOM, OPT), because those models' intermediate layers represent information in a rotated/shifted basis relative to the final unembedding coordinate system. The Tuned Lens (Belrose et al., 2023, arXiv:2303.08112) fixes this by learning a per-layer **affine probe** ("translator": $\hat h^{(l)} = A_l h^{(l)} + b_l$) trained to minimize KL divergence between the translated layer's decoded distribution and the model's own final-layer output distribution (not against ground-truth labels) — i.e. it is trained to predict what the *final layer* would say, making it a distillation-style calibration rather than a task probe.
- **Mechanism / derivation notes:**
  - Logit lens: $\text{logits}^{(l)} = W_U \cdot \mathrm{LN}_{\text{final}}(h^{(l)})$, applied at every layer $l$'s residual stream $h^{(l)}$, then softmax to get a vocabulary distribution "as decoded so far." Assumes the residual stream's basis is stable/shared across depth (true-ish for GPT-2, which nostalgebraist studied) — an assumption that breaks when models learn per-layer rotations/whitening the raw unembedding doesn't undo.
  - Tuned lens: train one affine map $(A_l, b_l)$ per layer $l$ (a linear "translator") on a frozen pretrained model so that $W_U \cdot \mathrm{LN}_{\text{final}}(A_l h^{(l)} + b_l)$ best matches (min KL) the model's actual final-layer output distribution, over a calibration dataset. This is an affine *probe* in the same family as a linear diagnostic classifier (Q1), but the training target is the model's own final belief rather than an external label — so it inherits the "probe accuracy ≠ causal use" caveat from Q1 in the same way (a well-calibrated tuned-lens decode at layer $l$ shows the final answer is *linearly recoverable* by that point, not that the residual stream literally "decides" there).
  - Framed as "iterative inference": each layer incrementally refines a distribution over outputs, with the (tuned) lens trajectory going from diffuse early-layer predictions to a sharp final distribution.
  - Reported downstream use: the trajectory of latent predictions elicited by the tuned lens can be used to detect anomalous/malicious inputs with high accuracy (an application beyond just visualization).
- **Values:**
  - GPT-Neo-2.7B: the logit lens's bias (KL divergence between the logit-lens marginal and the true final distribution) is "around 4 to 5 bits for most layers" (source: search-snippet summary of arXiv:2303.08112, not independently re-verified against the PDF table in this pass).
  - Tuned lens tested on autoregressive LMs "with up to 20B parameters" (i.e., includes GPT-NeoX-20B); tuned-lens-decoded perplexity is reported as uniformly lower and lower-variance across layers than logit-lens perplexity for Pythia and GPT-NeoX-20B, including when the final layer itself is excluded from the comparison (showing the improvement isn't just "collapsing to the true answer").
  - Logit lens originally introduced August 2020 (nostalgebraist, LessWrong post "interpreting GPT: the logit lens").
- **Sources:**

## Q3: World-model / structured-representation probes — Othello-GPT, linear vs. nonlinear

- **Finding:** Li et al. (2022) trained a GPT-style sequence model only on legal-move sequences of the board game Othello (no explicit board-state supervision) and found that a **nonlinear** probe (2-layer MLP) reading the model's internal activations could reconstruct the true board state (each of 64 squares: empty/black/white) with low error, evidence the model had built an internal "world model" of the game despite never being told the rules; a **linear** probe on the same raw activations performed poorly (high error) on this same task. Nanda et al. (2023) — first posted as the LessWrong piece "Actually, Othello-GPT Has A Linear Emergent World Representation" (Mar 2023), later published as "Emergent Linear Representations in World Models of Self-Supervised Sequence Models" (arXiv:2309.00941, BlackboxNLP 2023) — showed the *linear*-probe failure in Li et al. was an artifact of probing for the wrong featurization: the model does not represent squares as absolute black/white but as **relative to the current player to move** ("mine" vs. "theirs"); re-deriving probe labels in this player-relative frame makes a **linear** probe recover the board state with high accuracy, and the discovered linear directions can be used to causally **intervene** on the model's internal state (edit the activations) and change its predicted legal moves accordingly.
- **Mechanism / derivation notes:**
  - Task setup: model autoregressively predicts the next legal Othello move given move history; no board-state label is ever given as a training signal — any board-state information found via probing must be an emergent internal representation, not a supervised target.
  - Li et al. probe protocol: for each layer/position, train a classifier (linear, then 2-layer MLP) on frozen activations to predict the 3-way state (empty/black/white) of each of the 64 squares; report error rates. Nonlinear (MLP) probes achieve low error (searched snippet: error as low as ~1.7% in some condition), linear probes have high error on the same absolute black/white labeling.
  - Nanda's re-framing: relabel each square's probe target not as absolute color but as "belongs to the player whose turn it currently is" vs. "belongs to the opponent" (i.e., flip the color label's sign depending on whose move it is) — under this relabeling, a *linear* probe becomes highly accurate, showing the nonlinearity Li et al. needed was purely due to mismatched featurization (their linear probe had to learn an XOR-like flip based on move parity, which a linear classifier cannot do; a mine/theirs relabeling removes exactly that flip).
  - Causal validation: probe-derived direction(s) are used as an **intervention** vector — directly edited in the residual stream — and downstream legal-move predictions change consistent with the intended board-state edit, which is stronger evidence than passive probing alone (addresses the Q1 causal critique directly: this is probing *plus* an activation-patching-style causal check).
  - This is presented in the broader mech-interp literature as a key piece of evidence for the "linear representation hypothesis" — that models tend to represent meaningful features as linear directions in activation space once you find the right basis/featurization, even when a naive probe seems to show otherwise.
- **Values:** Li et al. (2022): nonlinear (2-layer MLP) board-state probe error reported as low as ~1.7% (vs. much higher error for a linear probe on the same absolute-color labeling — exact percentage not independently pulled from the PDF in this pass); legal-move generalization error 0.02% reported for one held-out/unseen-sequence condition (per search-snippet summary, not verified against the PDF table directly). Nanda et al. (2023): linear probe accuracy on player-relative ("mine/theirs") labeling reported as "high accuracy" in secondary summaries — exact percentage not pulled in this pass.
- **Sources:**
  - Li, K., Hopkins, A. K., Bau, D., Viégas, F., Pfister, H., Wattenberg, M. (2022/2023), "Emergent World Representations: Exploring a Sequence Model Trained on a Synthetic Task," ICLR 2023, arXiv:2210.13382 | tier=Primary | confidence=H
  - Nanda, N. (2023), "Actually, Othello-GPT Has A Linear Emergent World Representation," LessWrong, https://www.lesswrong.com/posts/nmxzr2zsjNtjaHh7x/actually-othello-gpt-has-a-linear-emergent-world (also https://www.neelnanda.io/mechanistic-interpretability/othello) | tier=Explainer (original finding) | confidence=H
  - Nanda, N., Lee, A., Wattenberg, M. (2023), "Emergent Linear Representations in World Models of Self-Supervised Sequence Models," BlackboxNLP 2023 (ACL Anthology 2023.blackboxnlp-1.2), arXiv:2309.00941 | tier=Primary (peer-reviewed follow-up of the LessWrong post) | confidence=H

## Q4: Reading attention responsibly — the "Attention is (not) Explanation" debate

- **Finding:** Jain & Wallace (2019, "Attention is not Explanation," NAACL 2019, arXiv:1902.10186) argue attention weights are a poor vehicle for explanation: across several NLP tasks they show (a) learned attention weights often correlate weakly with gradient-based feature-importance measures, and (b) one can construct **adversarial attention distributions** — very different attention weight patterns over the same input — that yield equivalent model output/predictions, i.e. the "explanation" the attention weights would suggest is not uniquely determined by the model's actual computation. Wiegreffe & Pinter (2019, "Attention is not not Explanation," EMNLP-IJCNLP 2019, arXiv:1908.04626) rebut this: they argue the definition of "explanation" matters, that a naive uniform-attention baseline and simple existence of *some* adversarial distribution that preserves output doesn't mean attention weights *actually used by the trained model* are uninformative, and propose four alternative tests (uniform-weights baseline comparison; variance calibration across random seeds; a frozen-weight diagnostic using pretrained representations; and an end-to-end-trained adversarial-attention model) — showing that when adversarial attention distributions are trained *end-to-end* (not just found post hoc), they are harder to construct and perform worse on their diagnostic than Jain & Wallace's post hoc construction suggested, i.e. naturally-trained attention is more constrained/informative than the original paper's adversarial-search framing implied. **Net implication for mech-interp practice:** raw attention-pattern inspection (e.g., "head H attends to token T, therefore the model is using T via this head") is not by itself a valid causal claim — it must be paired with an output/behavior-level check (ablation, patching, or an end-to-end faithfulness test) before being read as "the model does X because of this attention pattern," directly generalizing the Q1 probe-vs-causal-use gap to the attention-pattern case specifically.
- **Mechanism / derivation notes:**
  - Jain & Wallace's adversarial construction: post hoc, search for an alternate attention weight vector $\alpha'$ (different from the model's actual learned $\alpha$) subject to producing (near-)identical output logits, by exploiting the many-to-one map from (attention weights, values) to the weighted-sum output — i.e. the same output can be reached by very different weightings of the value vectors, showing non-identifiability of "the" explanatory attention pattern from output alone.
  - Wiegreffe & Pinter's rebuttal logic: the mere *existence* of an output-preserving alternate attention distribution (found by post hoc adversarial search, disconnected from training) does not mean the trained model's own attention weights are meaningless — the relevant test is whether an attention mechanism trained end-to-end under an adversarial objective (jointly optimized with the task loss to fool a diagnostic while preserving accuracy) can succeed; they find this is harder, arguing trained attention carries more real signal than the post hoc argument suggests.
  - Both papers pre-date and are scoped to encoder/RNN-style single-attention-layer architectures (e.g. LSTM+attention for classification/QA) common in 2019 NLP; scaling this debate to today's deep multi-head, multi-layer transformer LLM attention (with residual streams, many heads, and downstream MLP nonlinearities) is a nontrivial extrapolation — the modern mech-interp literature's response is generally to require causal-intervention evidence (activation patching, path patching, ablation) rather than resolve the debate by attention-weight inspection alone.
