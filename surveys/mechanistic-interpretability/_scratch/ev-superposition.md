## Q1: Toy Models of Superposition (Elhage et al. 2022) — exact toy setup and central claim

- **Finding:** Elhage et al. (2022), "Toy Models of Superposition" (Anthropic, transformer-circuits.pub), constructs a minimal "ReLU output model": $n$ sparse synthetic features are linearly projected down to $m < n$ hidden dimensions and back up, with a ReLU nonlinearity on the reconstruction, then trained to minimize an importance-weighted reconstruction loss. The central claim is that the trained network represents **more features than it has dimensions** by storing many features as (non-orthogonal, interfering) directions in the same low-dimensional space — "superposition" — and does so specifically when the input features are sparse; the ReLU acts as the nonlinear "interference filter" that makes this compression usable.
- **Mechanism / derivation notes:**
  - Setup: a synthetic dataset of feature vectors $\mathbf{x} \in \mathbb{R}^n$; each coordinate $x_i$ represents an independent, sparse, real-valued feature (nonzero with probability $1-S_i$, where $S_i$ is that feature's **sparsity**; equivalently "feature probability" $= 1 - S_i$).
  - Model ("ReLU output model"): a linear down-projection $W \in \mathbb{R}^{m\times n}$ ($m < n$) followed by the *transpose* back up (tied weights) plus bias and a ReLU nonlinearity applied only at the output:
    $x' = \mathrm{ReLU}(W^\top W x + b)$.
    (Intuition: $Wx$ is the "hidden activation" living in the bottleneck $\mathbb{R}^m$; $W^\top$ reconstructs; ReLU denoises/filters interference introduced by non-orthogonal feature directions.)
  - Loss: importance-weighted MSE, summed over features and averaged/summed over the data distribution:
    $L = \sum_{\mathbf{x}} \sum_i I_i\,(x_i - x'_i)^2$,
    where $I_i$ is a fixed **importance** weight for feature $i$ (a scalar multiplier on that feature's squared-error contribution — how much the network "wants" to represent that feature accurately). A common importance schedule used in the paper's experiments is geometric decay, e.g. $I_i = r^{\,i}$ for some $r<1$ (paper commonly uses values like $I_i=0.7^i$ per secondary-source restatement — verify exact constant against primary source before citing numerically).
  - Central result: when features are **dense** (low sparsity), the optimal solution learns to represent only the $m$ most important features exactly, along orthogonal directions (behaves like PCA / top-$m$ truncation), and ignores the rest. When features are **sparse** (sparsity high, i.e. features rarely co-occur), the network instead packs many more than $m$ features into the $m$-dimensional space as non-orthogonal, overlapping directions — this is superposition — trading a small amount of interference (cross-talk between non-orthogonal features when they do co-occur) for the ability to represent many more features than dimensions. The ReLU is what makes this trade favorable: it can suppress/zero out small negative interference terms rather than letting them corrupt the reconstruction.

- **Values:** $m < n$ (bottleneck dimension strictly less than feature count) — structural, not a specific number in the general setup; example importance schedule $I_i \approx 0.7^i$ (secondary-source paraphrase, low confidence on exact base — flag for verification); no other load-bearing numeric constants extracted this pass.

## Q2: Sparsity/importance phase diagram, feature geometry (antipodal pairs, regular polytopes), and capacity

- **Finding:** Elhage et al. (2022) show a sharp, phase-transition-like boundary in the (importance, sparsity) plane separating "no superposition" (each feature gets its own orthogonal dimension, or is dropped) from "superposition" (features share dimensions non-orthogonally). At low sparsity / high relative importance the model behaves like top-$m$ PCA; as sparsity increases past a threshold, the model transitions first into antipodal-pair packing, then into higher-order uniform geometric arrangements (regular polytopes), packing strictly more features than dimensions ("capacity" $> 1$ feature/dimension).
- **Mechanism / derivation notes:**
  - **Phase diagram.** Sweeping (feature importance ratio, feature sparsity) reveals a boundary: blue region = low sparsity / high relative importance → no superposition, each represented feature is orthogonal (or the feature is simply not represented at all if unimportant); red region = high sparsity / lower importance → strong superposition, many features packed into shared dimensions. The transition is described as "sharp," analogous to a physical phase transition.
  - **Antipodal pairs.** The first/simplest superposition structure: two features are assigned exactly opposite ($180°$) weight directions ($W_i = -W_j$), so they can share one dimension because they essentially never co-occur (sparse) — when one is "on" the other's negative direction contributes zero/negative pre-activation, which ReLU clips away, avoiding interference. E.g. three antipodal pairs packed together form an octahedron (6 features in 3 dimensions, each vertex a feature direction).
  - **Regular polytopes / uniform superposition.** As more features are forced into the same space at higher sparsity, the optimal weight geometry generalizes from antipodal pairs to uniform/regular polytope vertex arrangements that spread feature directions as evenly as possible on the unit sphere to minimize pairwise interference $(\hat W_i \cdot W_j)^2$ — explicitly identified structures: line segment (1D, 2 features), triangle and pentagon (2D), and octahedron / icosahedron-type arrangements to fit more features into 3D, generalizing to higher-dimensional uniform polytopes for larger feature counts. This is explicitly likened to the physics **Thomson problem** (uniform charge packing on a sphere) — points repelling each other on a sphere settle into the same kinds of symmetric/uniform configurations.
  - **Feature dimensionality metric.** For a learned weight matrix $W$ (columns $W_i$ per feature; $\hat W_i = W_i/\lVert W_i\rVert$), define per-feature dimensionality
    $D_i \triangleq \dfrac{\lVert W_i\rVert_2^2}{\sum_j (\hat W_j \cdot W_i)^2}$.
    $D_i = 1$ means feature $i$ has an entire dimension to itself (orthogonal to all others); $D_i = 0$ means the feature is not represented at all; a fractional $D_i$ (e.g. $D_i = 1/2$ for an antipodal pair) means the feature shares a dimension with others in superposition.
  - **Capacity / dimensions-per-feature.** Summing feature dimensionality over all features recovers (approximately) the bottleneck size: $\sum_i D_i \approx m$ (the model uses ~all of its available hidden capacity). An aggregate "dimensions per feature" statistic is defined via the Frobenius norm of the whole weight matrix, $\lVert W\rVert_F^2 = \sum_i \lVert W_i\rVert_2^2$, giving $D^\* \triangleq m / \lVert W\rVert_F^2$: a **lower** $D^\*$ means more input features are crammed per hidden dimension (more/denser superposition), a **higher** $D^\*$ (up to 1) means less superposition (closer to one feature per dimension). Conversely "features per dimension" $= 1/D^\*$ grows with sparsity; one secondary source states this growth follows approximately $1/(1-S)$ in $S$ (sparsity), with reported phase-transition landmarks at roughly features-per-dimension $\approx 1.7$ and $\approx 4$ marking, respectively, the onset of superposition and a further transition beyond simple antipodal packing into higher-order polytope regimes — flagged low-confidence pending direct primary-source confirmation of these two numeric landmarks.
- **Values:** $D_i = \lVert W_i\rVert_2^2 / \sum_j(\hat W_j\cdot W_i)^2$ (feature dimensionality, exact formula, M confidence); $\sum_i D_i \approx m$ (capacity ≈ full bottleneck usage); $D^\* = m/\lVert W\rVert_F^2$ (dimensions-per-feature); reported phase-transition landmarks "≈1.7" and "≈4" in features-per-dimension units (secondary-source paraphrase, confidence L — not yet confirmed against primary HTML figure/caption text); octahedron = 3 antipodal pairs in 3D (qualitative, confidence M).

## Q3: Polysemanticity vs monosemanticity; connection to sparse coding / compressed sensing

- **Finding:** A **monosemantic** unit (neuron or, more generally, a learned direction) activates for one clean, human-describable concept; a **polysemantic** unit activates for multiple unrelated concepts (e.g., one neuron responding to legal text, DNA sequences, and Hebrew script). The superposition hypothesis (Elhage et al. 2022) is presented as the dominant mechanistic explanation for polysemanticity: because the network represents more sparse features than it has neurons/dimensions, individual neurons end up as non-orthogonal linear combinations of many features' directions, so a single neuron's activation is driven by several unrelated underlying features at once. This is a central obstacle for mechanistic interpretability because it breaks the natural unit of study (the neuron) — you cannot read off "what this neuron means" if it is a superposed combination. The proposed fix (Bricken et al. 2023, "Towards Monosemanticity") is to train a sparse autoencoder (SAE) on the activations to learn an **overcomplete, sparse dictionary** of directions that individually recover monosemantic, human-interpretable features — i.e., algorithmically "unfold" the superposition back out into interpretable units, larger than the original activation space.
- **Mechanism / derivation notes:**
  - Polysemanticity ⇒ interpretability obstacle: standard circuit analysis assumes a neuron/channel is a meaningful unit; superposition means the "natural basis" (individual neurons/dimensions) is not privileged/interpretable, since features are stored as arbitrary (near-uniformly-spread, per Q2) directions rather than axis-aligned. This forces interpretability work to search for the right (over-complete) basis rather than reading off neurons directly.
  - **Explicit sparse-coding analogy** (toy-models paper, per convergent secondary sources): the toy-superposition setup is described as directly paralleling Olshausen & Field's classical sparse coding work on vision — in Olshausen & Field, sparsity is over *coefficients/activity* (each image uses only a few active basis coefficients out of an overcomplete dictionary, avoiding excessive interference); in toy-models superposition, sparsity is over *feature occurrence* (each input activates only a few of the many possible features), which is the mechanism that keeps collisions between non-orthogonal shared directions rare enough to tolerate.
  - **Explicit compressed-sensing analogy**: projecting a vector into a lower-dimensional space is normally non-invertible, but compressed sensing shows that if the original vector is known to be sparse, recovery from the low-dimensional projection is often possible under suitable "incoherence" conditions on the projection directions — mathematically the same regime the toy model exploits (sparse high-dim features → learned low-dim projection + nonlinear (ReLU) recovery step standing in for a compressed-sensing recovery algorithm). Both settings exhibit sharp phase transitions in recoverability as a function of the sparsity/dimension ratio.
  - **SAE-as-dictionary-learning fix**: Bricken et al. (2023) train a sparse autoencoder directly on transformer MLP activations, optimizing a reconstruction loss plus an $L_1$ sparsity penalty on the SAE's hidden ("feature") layer — i.e., solving sparse (overcomplete) dictionary learning post hoc on the superposed activations to recover an approximately monosemantic feature basis, each SAE-learned direction corresponding to a single interpretable concept, even though the original neuron basis was polysemantic.
  - Practical mitigations named across sources beyond SAEs: encouraging sparsity during training (e.g., $L_1$ regularization on activations), giving the model more neurons/capacity (e.g., Mixture-of-Experts architectures) so it needs less superposition, or purely post-hoc decoding via an overcomplete sparse-coding basis (the SAE approach).
- **Values:** Bricken et al. (2023) SAE experiment: 1-layer transformer, MLP width 512 neurons; SAE trained on **8B** MLP-activation datapoints; autoencoder (dictionary) sizes swept from **512 to 13,100** learned features (i.e., up to ~25× overcomplete relative to the 512-neuron MLP) — condition: "Towards Monosemanticity," one-layer transformer model, MLP activations.
