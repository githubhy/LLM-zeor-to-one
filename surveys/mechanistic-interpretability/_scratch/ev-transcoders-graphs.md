# Evidence: Transcoders, Crosscoders, Circuit Tracing / Attribution Graphs

## Q1: Transcoders (approximating an MLP sublayer with a wider sparse map from input to output, and why this eases circuit analysis vs SAEs)
- **Finding:** A transcoder is a sparse-dictionary model trained to reconstruct a component's *output* from its *input* (rather than reconstructing a layer's own activations from themselves, as an SAE does). Applied to an MLP sublayer, a transcoder replaces the dense, nonlinear MLP with a wider, sparsely-activating MLP-shaped map $\text{MLP}(x) \approx \text{TC}(x)$, whose hidden units are (ideally) monosemantic features. Because the transcoder's *input-side* encoder and *output-side* decoder are separate linear maps around a sparse nonlinearity, weights-based circuit analysis can trace a path directly from upstream features, through the MLP, to downstream features via matrix multiplication — the resulting attribution factorizes into an input-dependent term (which encoder features fired) and an input-invariant term (the fixed decoder-to-encoder weight path) — avoiding the intractable enumeration that plain-neuron or SAE-based analysis requires because SAEs only give you features *before and after* the MLP nonlinearity separately, not a linear path through it.
- **Mechanism / derivation notes:** Let MLP input be $x \in \mathbb{R}^d$, MLP output $y = \text{MLP}(x) \in \mathbb{R}^d$. A transcoder has encoder $W_{enc} \in \mathbb{R}^{d \times h}$ ($h \gg d$, e.g. 32x expansion), hidden pre-activation $z = \text{ReLU}(W_{enc}^\top x + b_{enc})$ (sparse, most entries zero), and decoder $W_{dec} \in \mathbb{R}^{h \times d}$ with $\hat y = W_{dec}^\top z + b_{dec}$, trained to minimize $\|\hat y - y\|_2^2 + \lambda \|z\|_1$ (reconstruction + sparsity penalty), exactly analogous to an SAE's loss but with target = MLP output instead of MLP input. Because $\hat y$ is a *linear* function of $z$ (decoder) and $z$'s support is determined by which sparse features fired on $x$, a circuit-tracing pass can treat "feature $i$ fires on this input, contributing $W_{dec,i}$ to the output" as a single edge, letting attribution graphs route through the MLP as if it were linear conditional on the active feature set — this is the "weights-based circuit analysis through MLP sublayers" the paper introduces. **Skip transcoders** (Paulo/Shabalin/Belrose 2025, and used in Anthropic's later attribution-graph work) add an affine skip connection $\hat y = W_{dec}^\top z + W_{skip} x + b_{dec}$ so the sparse hidden layer only needs to capture the nonlinear/interpretable residual of the MLP's function, not its whole affine component — this lowers reconstruction loss (a "Pareto improvement") with no cost to interpretability score, because the skip term still composes linearly into the attribution graph.
- **Values:** Original transcoder paper trained transcoders on models with 120M, 410M, and 1.4B parameters (Pythia-family sizes), reporting interpretability/faithfulness at least on par with SAEs at matched sparsity; applied to reverse-engineer the "greater-than circuit" in GPT2-small as a case study. Quote: "transcoders seek to faithfully approximate a densely activating MLP layer with a wider, sparsely-activating MLP layer" (arXiv:2406.11944 abstract, arxiv.org/abs/2406.11944). Skip-transcoder paper (arXiv:2501.18823): "skip transcoders, which add an affine skip connection to the transcoder architecture, and show that these achieve lower reconstruction loss with no effect on interpretability" — described as "a Pareto improvement on interpretability vs performance degradation" relative to plain transcoders and SAEs, across "all sizes" tested.

## Q2: Crosscoders (search: Lindsey, Templeton et al. 2024 Anthropic "Sparse Crosscoders for Cross-Layer Features and Model Diffing")
- **Finding:** A crosscoder is an SAE-like sparse dictionary model that reads from (and/or writes to) the activations of *multiple layers simultaneously* — instead of one SAE per layer, a single crosscoder learns one shared sparse feature dictionary whose latents can be active across several layers' residual streams at once. This directly targets cross-layer superposition/redundancy: the same conceptual feature often "persists" or gradually forms across several adjacent layers, and per-layer SAEs each re-discover a slightly-different, non-identical copy of it, fragmenting circuit analysis. Because the crosscoder's decoder has one weight block per layer sharing a common latent code, it can also be trained across *two different models* (e.g. a base model and its fine-tuned/chat variant) to find features shared between the models vs. features exclusive to one — the "model diffing" application — by comparing each latent's relative decoder norm between the two models' decoder blocks.
- **Mechanism / derivation notes:** Let $x^{(l)} \in \mathbb{R}^d$ be the residual-stream activation at layer $l = 1,\dots,L$ (or, for model-diffing, at the same layer of model A vs model B). A crosscoder has a single encoder mapping (a per-layer encoder block or a shared encoder, depending on variant) producing one shared sparse code $z = \text{ReLU}\!\big(\sum_l W_{enc}^{(l)\top} x^{(l)} + b_{enc}\big) \in \mathbb{R}^h$, and *per-layer* decoders $W_{dec}^{(l)}$ reconstructing each layer's activation from the same $z$: $\hat x^{(l)} = W_{dec}^{(l)\top} z + b_{dec}^{(l)}$. The training loss sums reconstruction error across all layers plus an L1 sparsity penalty on $z$, with the sparsity term scaled by the sum of per-layer decoder norms (so the penalty measures sparsity over (feature, layer) pairs, not just features) — this is what lets a feature "pay" only once for appearing at several layers, encouraging the model to reuse one latent rather than spawn near-duplicate per-layer features. For model diffing, the crosscoder replaces "layer $l$" with "model A vs model B" (same layer, different model), and each latent's **relative decoder norm** — $\|W_{dec}^{A}\|$ vs $\|W_{dec}^{B}\|$ for that latent — classifies it as shared (~0.3–0.7 relative norm, aligned decoder directions) vs exclusive to one model (near 0 or 1).
- **Values:** Crosscoders paper released Oct 25, 2024, authors Jack Lindsey*, Adly Templeton*, Jonathan Marcus*, Thomas Conerly*, Joshua Batson, Christopher Olah (Anthropic), transformer-circuits.pub/2024/crosscoders/index.html — framed explicitly as a preliminary "research update," not a full paper. Quote (search-snippet paraphrase, verify exact wording on fetch): "introduces sparse crosscoders, which read and write to multiple layers and produce shared features across layers and even models... particularly useful for model diffing and tracking how features move across layers or emerge after fine-tuning." Follow-up "Insights on Crosscoder Model Diffing" (Anthropic, Feb 2025, transformer-circuits.pub/2025/crosscoder-diffing-update/index.html and anthropic.com/research/crosscoder-model-diffing) reports that features *exclusive* to one model tend to be more polysemantic/dense (a competition-for-limited-capacity effect reproduced in toy models), and that practical value to date has mainly been shown in base-vs-fine-tune / base-vs-chat comparisons. Shared-latent relative-norm range "0.3 and 0.7" reported by a downstream replication/extension source (arXiv:2602.11729, "Cross-Architecture Model Diffing with Crosscoders" search-snippet) — needs primary-source (Anthropic 2024 post) confirmation, flagged below.

## Q3: Circuit tracing / attribution graphs (search: Anthropic 2025 "Circuit Tracing: Revealing Computational Graphs in Language Models" and "On the Biology of a Large Language Model", transformer-circuits.pub)
- **Finding:** Anthropic's March 2025 companion pair of papers — the methods paper "Circuit Tracing: Revealing Computational Graphs in Language Models" (transformer-circuits.pub/2025/attribution-graphs/methods.html) and the case-study paper "On the Biology of a Large Language Model" (transformer-circuits.pub/2025/attribution-graphs/biology.html) — trains a **cross-layer transcoder (CLT)** to replace every MLP sublayer in a model with a shared bank of sparse, (mostly) interpretable features, then builds a **local replacement model** for one specific input by swapping the real MLP computations for the CLT's linear feature paths (keeping attention patterns frozen at their original values and adding an "error node" per layer to absorb whatever the CLT failed to reconstruct). Because the local replacement model is linear in its active features (given frozen attention and fixed error terms), a direct Jacobian/attribution pass from output logits back through the graph produces an **attribution graph**: nodes = active CLT features (plus error nodes and input tokens), edges = the linear causal weight from one feature to another or to the output. The graph is then pruned (thresholding nodes/edges by influence) to a human-navigable size and explored interactively (open-sourced tooling, hosted via Neuronpedia). Applied to Claude 3.5 Haiku (Anthropic's Oct-2024 lightweight production model), the case-study paper reports internal multi-step ("two-hop") reasoning — e.g. representing "Texas" as an intermediate step when answering "capital of the state containing Dallas" → "Austin" — and forward/backward planning in poem-writing, where the model represents a candidate rhyme word (e.g. "rabbit") *before* writing the line that will end in it, and suppressing that internal rhyme representation causes the model to substitute a different rhyme (e.g. "habit") rather than fail to rhyme.
- **Mechanism / derivation notes:** (1) **CLT architecture**: a cross-layer transcoder takes the MLP *input* residual stream at layer $l$ and predicts a *contribution to the MLP output* not just at layer $l$ but additively at layer $l$ and every later layer $l' \geq l$ — i.e. it is a module taking a list of residual-stream inputs (one per layer) and returning a list of reconstructed residual-stream outputs, generalizing the single-layer transcoder of Q1 to let one sparse feature's effect "skip forward" across several MLP blocks at once, matching the fact that many MLP computations in real models are spread across multiple layers rather than confined to one. (2) **Local replacement model**: for a fixed input prompt, replace each MLP's exact nonlinear computation with the CLT's reconstruction (sparse features × per-layer decoder weights), but add a per-layer "error node" equal to (true MLP output − CLT reconstruction) so the replacement model's forward pass exactly reproduces the original model's output on that input — this is what makes the replacement "local" (input-specific, not a global approximation) and keeps the attribution faithful even where the CLT reconstruction is imperfect. Attention patterns (the softmax attention weights) are frozen at their real, input-computed values rather than linearized, because attention's nonlinearity is comparatively harder to sparsify/decompose than the MLP's. (3) **Attribution graph construction**: because the local replacement model's computation graph (features + frozen-attention edges + error nodes + embedding/unembedding) is now linear end-to-end, one can compute the direct effect of every non-zero feature/error-node/token on every other non-zero downstream feature and on the output logits via direct (one-hop) linear attribution, then chain these into a full graph via the chain rule (equivalent to a Jacobian-based backward trace); the graph is pruned by node/edge-influence thresholds to keep only the paths that matter for the traced behavior.
- **Values:** Both papers published March 27, 2025 by Anthropic's interpretability team, transformer-circuits.pub (methods: transformer-circuits.pub/2025/attribution-graphs/methods.html; biology: transformer-circuits.pub/2025/attribution-graphs/biology.html). Circuit-tracing tooling later open-sourced (Anthropic, "Open-sourcing circuit-tracing tools," anthropic.com/research/open-source-circuit-tracing), with an interactive frontend hosted by Neuronpedia and support for open-weight models (Gemma, Llama cited). Quote (paraphrase from search snippet, verify exact wording on fetch): "The CLT architecture consists of transcoders that take MLP inputs at each layer and predict MLP outputs for that layer and all subsequent layers." Case-study values: Claude 3.5 Haiku (Anthropic's lightweight production model, released Oct 2024) used as the subject model; the Dallas→Texas→Austin two-hop example and the rabbit/habit rhyme-planning example are both named as headline case studies (per Forbes/Moor Insights coverage and multiple secondary summaries of the biology paper).
